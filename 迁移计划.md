# 博客项目迁移计划

## 迁移概述

将博客项目从服务器 `1.94.119.167` 迁移到 `117.72.116.118`

## 源服务器信息

- **IP:** 1.94.119.167
- **项目路径:** `/home/DockerServes/`
- **MySQL 数据库:** `Shang-Blog`
- **MySQL Root 密码:** `root123`
- **数据大小:** MySQL 626MB，图片 17MB
- **Docker 镜像:** `myapp:latest`

## 迁移步骤

### 第一步：数据备份和导出

1. **导出 MySQL 数据库**
   ```bash
   docker exec mysql mysqldump -uroot -proot123 Shang-Blog > /tmp/shang-blog.sql
   ```

2. **导出 Docker 镜像**
   ```bash
   docker save myapp:latest -o /tmp/myapp.tar
   ```

3. **备份图片文件（可选）**
   ```bash
   tar -czf /tmp/upload.tar.gz /home/DockerServes/java/upload/
   ```

### 第二步：传输文件到目标服务器

1. **传输数据库备份**
   ```bash
   scp /tmp/shang-blog.sql root@117.72.116.118:/tmp/
   ```

2. **传输 Docker 镜像**
   ```bash
   scp /tmp/myapp.tar root@117.72.116.118:/tmp/
   ```

3. **传输图片文件（可选）**
   ```bash
   scp /tmp/upload.tar.gz root@117.72.116.118:/tmp/
   ```

### 第三步：目标服务器准备

1. **创建项目目录结构**
   ```bash
   mkdir -p /home/DockerServes/{mysql/{data/mysql,conf/conf.d},java/upload,nginx/{conf,conf.d,logs,html}}
   ```

2. **检查 MySQL 容器**
   - 如果目标服务器已有 MySQL，使用现有 MySQL
   - 如果没有，需要创建 MySQL 容器

### 第四步：导入数据

1. **导入 Docker 镜像**
   ```bash
   docker load -i /tmp/myapp.tar
   ```

2. **导入 MySQL 数据库**
   ```bash
   docker exec -i mysql mysql -uroot -p[密码] < /tmp/shang-blog.sql
   ```

3. **解压图片文件（如果传输了）**
   ```bash
   tar -xzf /tmp/upload.tar.gz -C /
   ```

### 第五步：配置和启动

1. **创建 docker-compose.yml**
   - 根据源服务器的配置创建
   - 调整网络配置以适应目标服务器

2. **启动服务**
   ```bash
   cd /home/DockerServes
   docker-compose up -d
   ```

3. **验证服务**
   - 检查容器状态
   - 测试数据库连接
   - 访问博客网站

## 注意事项

1. ⚠️ **MySQL 密码不同**：源服务器 MySQL root 密码是 `root123`，不是服务器密码
2. ⚠️ **网络配置**：目标服务器可能使用不同的 Docker 网络，需要调整
3. ⚠️ **端口冲突**：检查目标服务器端口是否被占用（3306, 8800等）
4. ⚠️ **数据一致性**：迁移期间建议停止源服务器服务，避免数据不一致
5. ⚠️ **图片文件**：用户表示图片可以重传，所以图片迁移是可选的

## 回滚方案

如果迁移失败，可以：
1. 保持源服务器运行
2. 恢复 DNS 指向源服务器
3. 清理目标服务器上的数据
