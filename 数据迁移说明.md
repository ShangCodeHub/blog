# 数据迁移说明

## 📦 数据目录结构

所有重要数据都映射到 `./docker/data/` 目录下，方便迁移和管理：

```
docker/data/
├── mysql/                    # MySQL 数据库
│   ├── data/                 # 数据库文件（重要：迁移时需要）
│   ├── conf/                 # MySQL 配置文件
│   ├── logs/                 # MySQL 日志
│   └── backup/               # 数据库备份目录
├── redis/                    # Redis 缓存
│   └── conf/                 # Redis 配置文件（数据不持久化）
├── upload/                   # 文件上传目录（重要：迁移时需要）
├── logs/                     # 应用日志
├── backend/                  # 后端配置
│   └── config/               # 后端配置文件（可选）
└── nginx/                    # Nginx 日志
    ├── web/                  # Web 前端日志
    │   └── logs/
    └── admin/                 # Admin 前端日志
        └── logs/
```

## 🔄 迁移步骤

### 方法一：完整目录迁移（推荐）

适用于完整迁移整个项目到新服务器。

#### 1. 停止服务
```bash
# 在源服务器上
cd docker
docker-compose down
```

#### 2. 打包数据目录
```bash
# 在源服务器上
cd docker
tar -czf blog-data-backup-$(date +%Y%m%d).tar.gz data/
```

#### 3. 传输到目标服务器
```bash
# 使用 scp 传输
scp blog-data-backup-20240203.tar.gz root@117.72.116.118:/root/shang/docker/

# 或使用其他传输方式（rsync、FTP等）
```

#### 4. 在目标服务器解压
```bash
# 在目标服务器上
cd /root/shang/docker
tar -xzf blog-data-backup-20240203.tar.gz
```

#### 5. 启动服务
```bash
# 在目标服务器上
cd docker
docker-compose up -d
```

### 方法二：选择性迁移

只迁移必要的数据，适用于部分迁移或数据清理。

#### 1. 迁移数据库
```bash
# 在源服务器上备份数据库
cd docker
docker exec shang-blog-mysql mysqldump -uroot -proot123 Shang-Blog > data/mysql/backup/backup.sql

# 传输到目标服务器
scp data/mysql/backup/backup.sql root@117.72.116.118:/root/shang/docker/data/mysql/backup/

# 在目标服务器上导入
cd docker
docker exec -i shang-blog-mysql mysql -uroot -proot123 Shang-Blog < data/mysql/backup/backup.sql
```

#### 2. 迁移上传文件
```bash
# 在源服务器上打包
cd docker
tar -czf upload-backup.tar.gz data/upload/

# 传输到目标服务器
scp upload-backup.tar.gz root@117.72.116.118:/root/shang/docker/

# 在目标服务器上解压
cd docker
tar -xzf upload-backup.tar.gz
```

#### 3. Redis 数据
```bash
# Redis 数据不持久化，重启后会自动清空
# 如果需要迁移 Redis 数据，可以在迁移前导出：
docker exec shang-blog-redis redis-cli --raw --pass redis123 --rdb /tmp/dump.rdb
# 然后在目标服务器导入（如果需要）
```

## 📋 迁移检查清单

迁移前检查：
- [ ] 已停止源服务器上的服务
- [ ] 已备份所有数据
- [ ] 已确认目标服务器 Docker 环境正常
- [ ] 已确认目标服务器端口未被占用（3307, 6380, 8800, 9001, 9003）

迁移后检查：
- [ ] 数据库连接正常
- [ ] Redis 连接正常
- [ ] 前端 Web 可以访问
- [ ] 前端 Admin 可以访问
- [ ] 文件上传功能正常
- [ ] 数据完整性验证

## 🔍 数据验证

### 验证数据库
```bash
# 进入 MySQL 容器
docker exec -it shang-blog-mysql mysql -uroot -proot123

# 检查数据库
SHOW DATABASES;
USE Shang-Blog;
SHOW TABLES;
SELECT COUNT(*) FROM sys_article;  # 检查文章数量
```

### 验证文件
```bash
# 检查上传文件
cd docker
ls -lh data/upload/
du -sh data/upload/
```

### 验证 Redis
```bash
# 进入 Redis 容器
docker exec -it shang-blog-redis redis-cli --raw --pass redis123

# 检查键数量（注意：Redis 数据不持久化，重启后会清空）
DBSIZE
KEYS *
```

## ⚠️ 注意事项

1. **数据一致性**
   - 迁移前确保停止所有服务，避免数据不一致
   - 建议在业务低峰期进行迁移

2. **权限问题**
   - 确保目标服务器上的目录权限正确
   - MySQL 数据目录需要正确的用户权限

3. **磁盘空间**
   - 确保目标服务器有足够的磁盘空间
   - 数据库文件可能较大，提前检查

4. **网络配置**
   - 如果目标服务器 IP 不同，需要更新前端配置中的 API 地址
   - 检查防火墙规则

5. **备份策略**
   - 迁移前务必做好完整备份
   - 保留备份文件直到确认迁移成功

## 🚀 快速迁移脚本

创建 `migrate.sh` 脚本（可选）：

```bash
#!/bin/bash
# 数据迁移脚本

BACKUP_NAME="blog-data-backup-$(date +%Y%m%d-%H%M%S)"
TARGET_HOST="117.72.116.118"
TARGET_PATH="/root/shang/docker"

cd docker

echo "1. 停止服务..."
docker-compose down

echo "2. 打包数据..."
tar -czf ${BACKUP_NAME}.tar.gz data/

echo "3. 传输到目标服务器..."
scp ${BACKUP_NAME}.tar.gz root@${TARGET_HOST}:${TARGET_PATH}/

echo "4. 在目标服务器执行以下命令："
echo "   cd ${TARGET_PATH}"
echo "   tar -xzf ${BACKUP_NAME}.tar.gz"
echo "   docker-compose up -d"

echo "迁移完成！"
```

## 📞 故障处理

如果迁移后出现问题：

1. **数据库无法启动**
   - 检查数据目录权限：`chown -R 999:999 data/mysql/data`
   - 查看 MySQL 日志：`docker-compose logs mysql`

2. **文件无法访问**
   - 检查文件权限：`chmod -R 755 data/upload`
   - 检查 Nginx 配置

3. **服务无法连接**
   - 检查网络配置：`docker network inspect shang-blog-network`
   - 检查服务状态：`docker-compose ps`
