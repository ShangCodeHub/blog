## 🚀 使用步骤

### 1. 构建后端 JAR
```bash
cd blog
mvn clean package -DskipTests
cp shang-server/target/*.jar SHANG-BLOG.jar
cd ..
```

### 2. 一键部署
```bash
# Windows
deploy.bat

# Linux/Mac
chmod +x deploy.sh
./deploy.sh
```

### 3. 访问服务
- 前端：http://localhost:9001
- 后端API：http://localhost:8800
- MySQL：localhost:3307（root/root123）
- Redis：localhost:6380（密码：redis123）

## 🔒 网络隔离说明

### 独立网络配置
- **网络名称**: `shang-blog-network`
- **子网**: `172.21.0.0/24`
- **网关**: `172.21.0.1`

### 服务 IP 分配
- MySQL: `172.21.0.2`
- Redis: `172.21.0.3`
- 后端: `172.21.0.10`
- 前端: `172.21.0.11`

### 与 Dify 隔离
- ✅ 使用不同的 Docker 网络
- ✅ 使用不同的端口映射
- ✅ 完全隔离，互不影响

## 📝 注意事项

1. **首次部署前**：
   - 确保已安装 Docker 和 Docker Compose
   - 先构建后端 JAR 文件
   - 检查端口是否被占用

2. **数据库迁移**：
   - 如果需要迁移旧数据，使用 `docker exec` 导入 SQL 文件
   - 图片文件可以手动复制到 `data/upload/` 目录

3. **配置修改**：
   - 修改 `docker-compose.yml` 中的环境变量
   - 修改 `application-pro.yml` 中的数据库配置
   - 修改 `nginx.conf` 中的代理配置

4. **生产环境**：
   - 修改默认密码（MySQL、Redis）
   - 配置 HTTPS（需要 SSL 证书）
   - 设置防火墙规则
   - 配置日志轮转
   - 设置自动备份
