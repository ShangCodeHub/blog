# 博客项目 Docker Compose 部署说明

## 📋 项目概述

本项目使用 Docker Compose 进行一键部署，包含以下服务：
- **MySQL 8.0.32** - 数据库服务（端口 3307）
- **Redis 7-alpine** - 缓存服务（端口 6380）
- **Spring Boot 后端** - Java 应用（端口 8800）
- **Vue Web 前端 + Nginx** - 用户端前端（端口 9001）
- **Vue Admin 前端 + Nginx** - 管理端前端（端口 9003）

## 🔒 网络隔离

- 使用独立的 Docker 网络 `shang-blog-network`（子网：172.21.0.0/24）
- 与服务器上其他应用（如 Dify）完全隔离
- 端口映射避免冲突：
  - MySQL: 3307（避免与 Dify 的 3306 冲突）
  - Redis: 6380（避免与 Dify 的 6379 冲突）
  - 前端Web: 9001（避免与其他服务冲突）
  - 前端Admin: 9003（避免与其他服务冲突）

## 🚀 快速开始

### 前置要求

1. **安装 Docker 和 Docker Compose**
   - Windows: 安装 Docker Desktop
   - Linux: 
     ```bash
     curl -fsSL https://get.docker.com | sh
     sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
     sudo chmod +x /usr/local/bin/docker-compose
     ```

2. **构建后端 JAR 文件**
   ```bash
   cd blog
   mvn clean package -DskipTests
   cp shang-server/target/*.jar SHANG-BLOG.jar
   cd ..
   ```

### 一键部署

#### Windows 系统
```bash
cd docker
deploy.bat
```

#### Linux/Mac 系统
```bash
cd docker
chmod +x deploy.sh
./deploy.sh
```

#### 手动部署
```bash
# 1. 进入 docker 目录
cd docker

# 2. 创建必要的目录（脚本会自动创建）
mkdir -p data/mysql/{data,conf,logs,backup}
mkdir -p data/redis/conf
mkdir -p data/upload
mkdir -p data/logs
mkdir -p data/backend/config
mkdir -p data/nginx/{web,admin}/logs

# 3. 构建并启动服务
docker-compose up -d --build

# 4. 查看服务状态
docker-compose ps

# 5. 查看日志
docker-compose logs -f
```

## 🔧 配置说明

### 端口配置

| 服务 | 容器端口 | 主机端口 | 说明 |
|------|---------|---------|------|
| MySQL | 3306 | 3307 | 避免与 Dify 冲突 |
| Redis | 6379 | 6380 | 避免与 Dify 冲突 |
| 后端 | 8800 | 8800 | Spring Boot 应用 |
| 前端Web | 80 | 9001 | Nginx 静态文件（用户端） |
| 前端Admin | 80 | 9003 | Nginx 静态文件（管理端） |

### 网络配置

- **网络名称**: `shang-blog-network`
- **子网**: `172.21.0.0/24`
- **网关**: `172.21.0.1`
- **服务 IP**:
  - MySQL: `172.21.0.2`
  - Redis: `172.21.0.3`
  - 后端: `172.21.0.10`
  - 前端Web: `172.21.0.11`
  - 前端Admin: `172.21.0.12`

### 环境变量

后端应用通过 `application-pro.yml` 配置连接数据库：
- MySQL: `jdbc:mysql://mysql:3306/Shang-Blog`
- Redis: `redis:6379`

注意：在 Docker Compose 中，服务名（如 `mysql`、`redis`）会自动解析为对应的容器 IP。

## 📝 常用命令

### 服务管理

```bash
# 进入 docker 目录
cd docker

# 启动服务
docker-compose up -d

# 停止服务
docker-compose down

# 重启服务
docker-compose restart

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f [服务名]  # 如: docker-compose logs -f backend
```

### 数据库操作

```bash
# 进入 MySQL 容器
docker exec -it shang-blog-mysql mysql -uroot -proot123

# 导入数据库（如果需要）
docker exec -i shang-blog-mysql mysql -uroot -proot123 Shang-Blog < shang-blog.sql

# 备份数据库
docker exec shang-blog-mysql mysqldump -uroot -proot123 Shang-Blog > backup.sql
```

### 镜像管理

```bash
cd docker

# 重新构建镜像
docker-compose build --no-cache

# 查看镜像
docker images | grep shang-blog

# 删除镜像
docker rmi shang-blog-backend shang-blog-frontend-web shang-blog-frontend-admin
```

## 🔍 故障排查

### 1. 服务无法启动

```bash
# 查看详细日志
docker-compose logs [服务名]

# 检查端口是否被占用
netstat -an | grep -E "3307|6380|8800|9001"

# 检查 Docker 网络
docker network ls
docker network inspect shang-blog-network
```

### 2. 数据库连接失败

```bash
# 检查 MySQL 容器是否正常运行
docker-compose ps mysql

# 检查 MySQL 日志
docker-compose logs mysql

# 测试数据库连接
docker exec -it shang-blog-mysql mysql -uroot -proot123 -e "SHOW DATABASES;"
```

### 3. 前端无法访问后端 API

- 检查 `blog-web/nginx.conf` 或 `blog-admin/nginx.conf` 中的代理配置
- 确认后端服务正常运行：`docker-compose ps backend`
- 查看 Nginx 日志：
  - Web前端：`docker-compose logs frontend-web`
  - Admin前端：`docker-compose logs frontend-admin`

### 4. 文件上传问题

- 检查 `data/upload` 目录权限
- 确认后端配置的 `UploadPath` 路径正确
- 查看后端日志：`docker-compose logs backend`

## 📊 数据备份与恢复

### 备份

```bash
cd docker

# 备份 MySQL 数据库
docker exec shang-blog-mysql mysqldump -uroot -proot123 Shang-Blog > data/mysql/backup/backup-$(date +%Y%m%d).sql

# 备份上传文件
tar -czf data/upload-backup-$(date +%Y%m%d).tar.gz data/upload/

# 完整备份所有数据（推荐，方便迁移）
tar -czf blog-data-backup-$(date +%Y%m%d).tar.gz data/
```

### 恢复

```bash
cd docker

# 恢复数据库
docker exec -i shang-blog-mysql mysql -uroot -proot123 Shang-Blog < data/mysql/backup/backup-20240203.sql

# 恢复上传文件
tar -xzf data/upload-backup-20240203.tar.gz

# 完整恢复所有数据
tar -xzf blog-data-backup-20240203.tar.gz
```

### 数据迁移

所有重要数据都映射到 `./data/` 目录，方便迁移：
- **数据库**: `./docker/data/mysql/data` - 直接复制此目录即可迁移数据库
- **上传文件**: `./docker/data/upload` - 直接复制此目录即可迁移所有上传文件
- **注意**: Redis 数据不持久化，重启后会自动清空，通常不需要迁移

详细迁移说明请查看：[数据迁移说明.md](./数据迁移说明.md)

## 🛡️ 安全建议

1. **修改默认密码**
   - MySQL root 密码：修改 `docker-compose.yml` 中的 `MYSQL_ROOT_PASSWORD`
   - Redis 密码：修改 `docker-compose.yml` 和 `data/redis/conf/redis.conf`

2. **限制端口访问**
   - 在生产环境中，建议使用防火墙限制端口访问
   - 只开放必要的端口（如 9001 前端端口）

3. **定期备份**
   - 设置定时任务自动备份数据库和文件

4. **监控日志**
   - 定期查看应用日志，及时发现异常
